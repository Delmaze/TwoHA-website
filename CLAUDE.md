# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

TwoHA - 앱 출시를 위한 회사 홈페이지 제작 프로젝트. 웹 기반 정적/동적 홈페이지.

## 핵심 규칙 (위반 시 작업 중단)

- **20줄 이상 출력 예상 시**: context-mode MCP 도구 사용 필수. Bash/Read 직접 호출 금지.
- **코드 작업 응답 형식**: 코드/설정 변경이 포함된 모든 작업은 5단 형식 필수 — 1.원인 2.수정 3.재발방지 4.검증 5.리스크. 미준수 시 작업 완료로 간주하지 않음.
- **파일 탐색 순서**: `qmd` 먼저 검색 → 결과 없을 때만 Read/Glob/Grep 허용.
- **이해 후 작업**: 95% 이상 이해된 경우에만 수정 시작. 불분명한 부분은 질문.
- **요청 범위 준수**: 요청하지 않은 리팩토링·네이밍·구조 개편 금지. 개선점은 제안만.
- **검증 없이 완료 보고 금지**: 가능한 검증을 실제로 수행한 후 보고. 불가능하면 "검증 불가 (사유)" 명시.

## 작업 규칙

- **커밋**: 각 작업 완료 후 git commit 필수. 커밋 메시지는 한글로, prefix 사용 (feat/fix/chore/docs/refactor).
- **문서 업데이트**: 작업 완료 후 관련 docs/plan 문서에 변경 내용 및 현재 상태 정리.
- **주석**: 코드 작성 시 수정 목표 주석 남길 것.

## 응답 5단 형식 상세 (코드 변경 작업 한정)

| 칸 | 내용 |
|----|------|
| 1. 원인 | 문제의 근본 원인 |
| 2. 수정 | 변경 내용 |
| 3. 재발방지 | 근본 원인 수정 / 가드레일 / 회귀 테스트 / 실패 가시성 로깅 중 하나 이상. 핫픽스면 "의도적 생략 (사유)" 명시 |
| 4. 검증 | 실제 실행한 것만 기록. 화이트리스트(dev서버, 유료 API, 브라우저 e2e)는 "검증 불가"로 충분 |
| 5. 리스크 | 회귀 가능성·미검증 영역만. wishlist·후속 개선 금지. 없으면 "해당 없음" |

## MCP 도구 사용

| 도구 | 용도 |
|------|------|
| `ctx_execute` | CLI 명령 실행, API 호출 |
| `ctx_execute_file` | 파일 분석 (탐색 목적) |
| `ctx_fetch_and_index` + `ctx_search` | 외부 문서 참조 |
| `ctx_batch_execute` | 여러 명령 + 쿼리를 한 번에 실행 |
| `qmd` | 코드 탐색 시 최우선 검색 (`.js`, `.html`, `.css`, `.md`) |
| `context7` | 라이브러리/프레임워크 공식 문서 참조 |

- 파일 **편집** 목적으로 읽을 때는 Read 도구 사용 (Edit에 필요)
- Bash는 git, mkdir, rm, mv 등 단순 명령 전용

## 효율 규칙

- 이미 읽은 파일 재확인 금지.
- 가능한 도구 호출 동시 실행.
- 사용자가 이미 설명한 내용 반복하지 말 것.
